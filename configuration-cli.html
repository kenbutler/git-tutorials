Configuration

User Identity

View the user’s name and email the Git config is storing:

git config user.name

git config user.email

Using these commands, you can view what Git currently believes your name and email are. Sometimes what Git shows, if anything, is not always right though, and we need a way to change that.

Let’s play out a scenario.

Configuration: User Identity

This developer's current Git user identity is shown below. She wants to correct this, though, to be the actress's (or...developer's) name rather than her character.

Name: Princess Leia

    ---> Carrie Fisher

Email: Princess.Leia@gmail.com

    ---> Carrie.Fisher@gmail.com

1.) Update the user's name

git config user.name "Carrie Fisher"

2.) Update the user's email

git config user.email "Carrie.Fisher@gmail.com"

This will update the user's Git identity locally, meaning only for the current repository she is working with. If she wanted to update it for all repositories on her computer, she could add the `--global` flag, like so:

git config --global user.email "Carrie.Fisher@gmail.com"

This initial configuration is important for documentation. Using the command `git log`, anyone can look back through the history of commits and see who made a change, when, what about, and how to contact that user.

Note that changing your user identity will not retroactively change details of a previous commit message, as seen in the example below. 

                
                    $ git log
                    commit e73f0226c6d4d0229d35140cbee96074de485299 (HEAD -> master)
                    Author: Carrie Fisher <Carrie.Fisher@gmail.com>
                    Date:   Wed Dec 5 23:40:10 2021 -0500

                        Created distress message to Obi-Wan Kenobi.

                    commit 102b65127502a3e57319c77acc213fe30cd16d96
                    Author: Princess Leia <Princess.Leia@gmail.com>
                    Date:   Wed Dec 4 12:22:02 2021 -0500

                        Added documentation regarding escape plan.

Diff and Merge Tools

When using Git, you’ll quickly realize that comparing different versions of a file or merging changes together can get a bit tedious without another tool. There are a variety of tools on the Internet that can help you tremendously in viewing different versions of a file, handling Git conflicts, and more. You’ll likely want to hook one of these into your Git configuration.

Configuration: Diff and Merge Tools

Three, common tools for this are: BeyondCompare, Meld, and KDiff3
Check out a larger ranking of available tools as of 2018.



Configuration: Diff Tools

Diff tools allow us to view differences between commits. This can be especially useful when determining the cause of a bug. Git will choose a difftool by default, if it is not set. Here we can see how to set your configure Meld as your difftool. Place this code within your .gitconfig file in your home directory. 

                
                    [diff]
                        tool = meld     # This sets the tools
                    [difftool]
                        prompt = false  # This prevents a confirmation prompt each time we try to open it
                    [difftool "meld"]   # This sets command for opening the tool. Dependent on OS
                        trustExitCode = true
                        cmd = open -W -a Meld --args \"$LOCAL\" \"$PWD/$REMOTE\"

To use this, you can simply enter git difftool on the command line to view your current changes. If you want to inspect changes between two commits, use the following format:

git difftool [starting abbreviated commit hash] [ending abbreviated commit hash]

Configuration: Merge Tools

Merge tools allow us to visually resolve merge conflicts. As before, Git will select a default mergetool if one is not set. The setup for these is nearly identical to diff tools. Place this code within your .gitconfig file in your home directory. 

                
                    [merge]
                        tool = meld     # This sets the tools
                    [mergetool]
                        prompt = false  # This prevents a confirmation prompt each time we try to open it
                    [mergetool "meld"]   # This sets command for opening the tool. Dependent on OS
                        trustExitCode = true
                        cmd = open -W -a Meld --args --auto-merge \"$PWD/$LOCAL\" \"$PWD/$BASE\" \"$PWD/$REMOTE\" --output=\"$PWD/$MERGED\"

You can use mergetool in the same way that you use difftool.

Aliases

Git aliases are similar in nature to regular aliases one might set in a Bash profile. We can use them to save complicated Git commands in a concise form. Below are some that I have found useful.

(These belong in the .gitconfig file in your home directory.) 

                    
                        [alias]
                            hist = log --pretty=format:\"%h %ad | %s%d [%an]\" --graph --date=iso
                            histf = log --pretty=format:\"%h %ad | %s%d [%an]\" --graph --date=iso --name-status
                            histc = log --graph --pretty=oneline --abbrev-commit

Click the down arrow for explanations of each of these flags
Further details and more git log flags can be found here

Configuration: Aliases -- Flags Explained

--pretty
If specified as `oneline`, this will condense the log to a single line. Otherwise, you can specify the format of the log message you desire. Format placeholders are found here (search for 'format:<string>')

--graph
This options draws a text-based graphical representation of the commit history on the left hand side of the output. Select the down arrow for how this looks with `git histc`.

--date
This controls how the date timestamp is presented in the log message.

--name-status
This contents the log message to show only the name and status of changed files.

--abbrev-commit
Instead of showing the full 40-byte hexadecimal commit object name, show a prefix that names the object uniquely.
