Installation: SourceTree

Important Note

SourceTree is currently only available for Windows and Macintosh systems.
The creator of SourceTree, Atlassian, has openly stated they are not pursuing compatibility with Linux-based systems.

Installation: SourceTree


SourceTree is an Atlassian product. Atlassian realized that not all of its users would be comfortable with using the command line, so they provided a clean, graphical user interface to help users manage repositories.

Essentially SourceTree is a desktop app that helps you use Git functionality with your repositories.

Installation: SourceTree

A Common Misconception

Visiting GitHub, Bitbucket, or other Git servers in your web browser will not allow you to clone down a copy of your repository. Bitbucket, for example, is merely an interface for viewing elements of your repository. SourceTree offers methods for cloning.

It is important to be truthful and accurate in your user identity configuration
This is your Git configuration and marks your commits with your name and email. As an convenient option, you may also select the checkbox to determine whether or not these credentials should be globally applied to all repositories on your computer.

After user identity configuration...

If you connected to a server, you will be brought to the "Remote" tab by default, as this image shows. You should be able to see all of the repositories available to you on the remote server (i.e. the same you would see by logging into the associated website, such as GitHub or Bitbucket).

If you did not connect to a server, you may be on the Local tab with an empty repository list.

You can see that every remote project here has a ”Clone” button to the right of it. You can use this button to clone down repositories.

In order to successfully clone down a repository, you'll need to ensure the security protocol is configured correctly between your computer and the repository server.

If you have already cloned down a repository, you will see the branch that you have checked out. We will see this shortly.

When you select "Clone", SourceTree will ask you to specify a destination path on your local computer and a name of a directory to place it in. Typically leaving the defaults in place makes the most sense.

Once you’ve cloned down a repository, you’re ready to get to work!

If you select the ”Local” tab, you will see any repositories that you have
(a) cloned down, or
(b) created locally and hooked into SourceTree.

Local repositories may only be local to your machine, whereas remote repositories are guaranteed to exist on the server.

All of your local repositories -- either from cloning down or creating in isolation -- will indicate which branch is currently checked out.

Binary Files and SourceTree
